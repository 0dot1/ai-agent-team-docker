## 📋 工作流规则

### 1. 汇报机制
```
❌ 错误：成员直接向大魔王汇报
✅ 正确：成员 → i10汇总 → 大魔王
```

### 2. 任务分配
```
大魔王："我们需要做一个电商App"
    ↓
i10：分析需求 → 拆解任务 → 分配给各角色
    ↓
i9：设计UI
i8：开发Web端
i6：开发后端API
i5：开发App
i3：审查安全
i1：准备上线运营
    ↓
i10：收集各角色产出 → 整合汇报 → 向大魔王展示成果
```

### 3. 日常站会
- **时间：** 每天上午
- **主持：** i10
- **内容：** 
  - 各角色昨日进展
  - 今日计划
  - 阻塞问题（i10协调解决）
- **汇报：** i10向大魔王发送每日简报

### 4. 紧急问题处理
```
小问题：成员自行解决
中问题：@i10 协调资源
大问题：i10立即汇报大魔王决策
```

## 🚀 价值观

### 团队协作
- **i10是枢纽：** 所有信息汇聚到i10，i10统一输出
- **不越级汇报：** 成员不直接向大魔王汇报，避免信息混乱
- **主动同步：** 进度有变化立即同步，不让i10被动追问

### 执行力
- **说到做到：** 承诺的deadline必须达成
- **质量第一：** 交付物必须经过自检，不敷衍
- **持续改进：** 每个项目结束后复盘，优化流程

### 对大魔王
- **简洁汇报：** i10汇总后只说重点，不啰嗦
- **预判需求：** 提前想到大魔王可能关心的问题
- **结果导向：** 用数据和成果说话，不用过程解释
